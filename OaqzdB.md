javac 是 Java 编译器命令用于将编译器命令 .java 源文件编译为 JVM 可执行的 .class 字节码文件。掌握常用参数可以提高开发效率，控制编译行为，适应不同的环境。

指定输出目录（-d）
编译生成的 .class 默认将文件放在源文件所在目录中使用 -d 可以指定输出根目录，建议与包结构一起使用。

若源文件含 package com.example;，执行 javac -d ./out com/example/Hello.java，则 Hello.class 会生成在 ./out/com/example/ 下
  当目录不存在时，javac 不会自动创建父目录(如 -d ./out/a/b 而 ./out 不存在，会报错)，需要提前创建或配合 mkdir -p（Linux/macOS）或 mkdir /p（Windows）

设置源代码和目标字节码版本（-source 和 -target）
用于与旧版兼容 JDK 或限制语言特征，避免低版高版语法 JVM 上运行失败。


-source 8：允许使用 Java 8 语法（如 Lambda），但禁止 Java 9+ 特性(如模块声明)
  
-target 8：生成的 .class 文件版本号为 52（对应 Java 8），可在 Java 8 及更高版本 JVM 运行
  注意：-source 和 -target 应匹配或 -target ≤ -source；默认情况下不加时 JDK 版本处理

指定类路径（-cp 或 -classpath）
告诉编译器在哪里可以找到被引用的类(例如第三方) JAR 或自定义类)，特别是在编译依赖其他模块的代码时。
；



格式：javac -cp &quot;lib/commons-lang3.jar:src&quot; Main.java（Linux/macOS，用冒号分隔)
  Windows 写法：javac -cp &quot;lib\commons-lang3.jar;src&quot; Main.java（用分号）
  可用于当前目录 . 表示，如 -cp &quot;.:lib/*&quot;（Java 6+ 支持通配符 * 匹配 JAR，但不递归子目录)

使用详细的输出和警告（-verbose、-Xlint）
特别适用于辅助排查和编译问题 CI 在环境或团队合作中统一代码规范。


-verbose：类加载、输入输出文件等详细信息在打印和编译过程中
  
-Xlint：打开所有编译期警告(如未使用变量、过时 API、空指针隐患等))
  具体的警告类型可以指定，例如， -Xlint:deprecation(仅报告已弃用 API 使用）、-Xlint:unchecked(与泛型类型擦除有关)
  搭配 -Werror 所有警告都可以视为错误，只有在强制修复后才能编译

多个参数在实际使用中经常组合，如：javac -source 11 -target 11 -d ./bin -cp &quot;.:lib/*&quot; -Xlint:all -Werror src/com/example/App.java

虽然不复杂，但很容易被忽视 —— 参数顺序一般不影响结果，但是 -d 和 -cp 之后必须跟上路径，当路径有空间时，需要用英文引号包裹。
