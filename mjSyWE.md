Java版本选择哪一个？JDK 17还是JDK 21？
Java本地开发环境稳定性的第一个门槛是JDK版本的选择。JDK 17（LTS）和JDK 21（LTS）现在都有长期的支持，但是JDK在实际项目中 17更安全——大多数主流框架（Spring Boot 3.0+完全适合JDKK 21虚拟线程等特性)，Spring Boot 2.7.x在JDKK中仍然广泛应用于股票项目 21下可能触发UnsupportedClassVersionError或IncompatibleClassChangeError。
建议根据以下条件做出决定：

Springg Boot 3.2+、Micrometer 1.12+等新版生态生态新版生态版 → 选jdk-21.0.3(推荐Adoptiumm 构建Temurin)
维护旧系统，对接遗留的中间件(如Weblogic) 14c、IBM JDK兼容组件)→ 锁定jdk-17.0.8

避免使用Oracle JDK商业版(自JDK) 17起需要付费商用)，改用temurin-17.0.8_7-jdk或zulu-21.32.17-ca-jdk

不要混用不同厂商的JDK(比如IntelliJ用Temurin，Maven用Zulu），会导致javac与java行为不一致

PATH和JAVA_HOME如何设置不冲突？
Windows/macOS/Linux上最常踩的坑不是装错JDK，而是环境变量污染。典型现象:终端java -version显示JDK 但IDE报错了“21”Module not found: java.base或者Maven编译提示source release 21 requires target release 21——本质是JAVA_HOME没对齐，或PATH旧JDK路径残留在里面。
正确做法：
；


JAVA_HOMEJDK根目录必须指向(不是)jre/例如，子目录)/Library/Java/JavaVirtualMachines/temurin-17.0.8+7/Contents/Home（macOS）

PATH中只保留$JAVA_HOME/bin，删除任何绝对路径/usr/bin/java或C:\Program Files\Java\jdk1.8.0_202\bin

Shell配置文件（~/.zshrc或~/.bash_profile）里用export JAVA_HOME=...，别用set JAVA_HOME=...(后者是Windows cmd语法)
IDE（IntelliJ/Eclipse）SDK需要单独配置，不继承系统JAVA_HOME；依赖Mavenmvn -vJava版本的输出，而不是IDE设置

Maven本地仓库卡住或验证失败怎么办？
本地开发环境“不稳定”的高频诱因是Maven依赖拉取异常:下载中断SHA256 checksum failed、Could not transfer artifact。这种情况通常不是网络问题，而是当地仓库元数据损坏或镜像源配置失效。
			
		
快速修复步骤：

确认~/.m2/settings.xml中指向可用镜像(国内推荐)https://maven.aliyun.com/repository/public，不要使用已停止的服务http://maven.oschina.net）
例如，删除错误模块的本地缓存目录rm -rf ~/.m2/repository/org/springframework/boot/spring-boot-starter-web，再重跑mvn clean compile

禁用校验(仅临时调试)：mvn compile -Dmaven.wagon.http.ssl.insecure=true -Dmaven.wagon.http.ssl.allowall=true

避免手动修改~/.m2/repository下的_remote.repositories或maven-metadata-central.xml——这些文件由Maven自动维护，手改必须混乱

为什么IntelliJ中的模块编码和编译输出路径总是出错？
即使JDK和Maven正常，IntelliJ仍然可能无法编译，错误如Cannot resolve symbol 'List'或Output path is not specified。根本原因是项目级配置与模块级配置不对齐。
关键检查点：

File → Project Structure → Project：确认Project SDK和Project language level匹配(例如SDK选择17，language level不能设为8）
File → Project Structure → Modules → Sources tab：确保src/main/java标记为Sources，且src/test/java标记为Test Sources；否则，该类将不会被编译器识别
File → Settings → Build → Compiler → Java Compiler：勾选Use compiler from module target bytecode version，否则，JDK可能会产生 JDK用于8字节码 17运行
Build → Build Artifacts → Output 如果directory是空的，会直接导致ClassNotFoundException，需要手动指定为$PROJECT_DIR$/out/artifacts


环境稳定的核心不是“安装更多”，而是在“正确”-JDK版本、环境变量、仓库配置和IDE参数中必须形成闭环。在CI构建或同事合作过程中，任何链接都将以默认值携带。
