ArrayList基于数组，查询速度快，删除速度慢；LinkedList基于链表，首尾增删速度快，但随机访问速度慢；Vector线程安全但性能低已过时；CopyonWritearayList读多写少并发安全。

ArrayList：基于数组的动态列表
ArrayList 是最常用的 List 实现，底层使用 Object 数组存储元素，支持随机访问，快速查询（O(1)增删慢(尤其是中间位置，需要移动元素，平均 O(n)）。线程不安全，但性能高，适用于读多写少、按索引频繁访问的场景。
使用建议：
；

初始化时可指定容量，避免多次扩容(默认初始容量 10、扩容为原长 1.5 倍）
  如果已知一般元素的数量，建议推荐 new ArrayList(expectedSize)，减少 resize 开销
  在全历时优先使用增强 for 避免在循环中调用或迭代器 remove(int index) 引发 ConcurrentModificationException(除非使用迭代器 remove()）

LinkedList：基于双向链表的列表
LinkedList 底层是双向链表节点，不支持真正意义上的随机访问（get(i) 需要从头到尾遍历，O(n)），但是，首尾插入/删除非常快（O(1)。同时实现了 Queue 和 Deque 接口，可作为栈、队列使用。
使用建议：
；

只在头部或尾部频繁增删，很少按索引检查元素时选择
  避免用 get(i) 性能远远低于通历或随机访问 ArrayList
  如需栈行为，使用 push/pop；需要队列行为，使用 offer/poll；更语义化、更安全

Vector：线程安全但过时的动态数组
Vector 和 ArrayList 结构相似(数组实现)，但全部结构相似 public 方法都加了 synchronized，因此，线程是安全的。但同步粒度粗，性能低 API 陈旧的设计(如 elementAt()、firstElement() 等）。
			
		
使用建议：
；

不建议使用新项目 Vector，除非维护遗留代码
  如果需要安全的线程 List，优先考虑 Collections.synchronizedList(new ArrayList())，或根据场景选择 CopyOnWriteArrayList(适用于阅读远多于写作的并发场景)
  注意：synchronizedList 只保证单个操作的原子性，复合操作(如检查后插入)仍需手动同步

CopyOnWriteArrayList：读多写少的并发列表
每次在底层写操作（add、set、remove）复制整个数组，阅读操作无锁，完全并发安全。适用于阅读频率极高、写作少(如监听器列表、配置缓存)的场景。
使用建议：
；

写作操作费用大(复制数组) + GC 严禁高频修改场景的压力)
  迭代器是基于快照的，即使列表在遍历时被修改，也不会被抛弃 ConcurrentModificationException，但迭代开始后没有变化
  不支持 null 元素（add(null) 会抛 NullPointerException）
