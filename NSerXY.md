JVM内存分为两类:线程私有(程序计数器、虚拟机栈、本地方法栈)和线程共享(堆叠、方法区/元空间、运行时常量池、直接内存)；前者随线程生灭，无数据竞争，后者由JVM管理，需要注意同步和GC。

JVM 根据线程共享，内存区域分为两类：线程私有和线程共享。理解这一划分是分析内存泄漏GC 行为和多线程问题的关键。

线程私有区域：每个线程独占一份
这类区域随着线程的创建而分布，随着线程的结束而销毁，不与其它线程共享，因此不存在数据竞争问题。


程序计数器（PC Register）：记录当前线程执行的字节码行号。唯一不会发生的 OutOfMemoryError 的区域。
  
Java 虚拟机栈（Java Virtual Machine Stack）：存储方法调用的栈帧包括局部变量表、操作数栈、动态链接、方法出口等。每种方法调用相应的栈帧进入栈，返回栈。
  
本地方法栈（Native Method Stack）：为 JVM 调用 Native 方法（如 JNI）服务类似于虚拟机栈，可以与虚拟机栈合并(如 HotSpot 将两者合一)。

线程共享区域：所有线程共享相同的份额
这些区域由 JVM 创建起始时，生命周期和 JVM 同样，多个线程可以同时访问数据，需要注意同步和可见性。


堆（Heap）：存储对象的实例和数组是 GC 主要管理区域。几乎所有通过。 new 所有创建的对象都在这里分配(逃逸分析优化除外)。
  
方法区（Method Area）：存储虚拟机加载的信息、常量、静态变量、即时编译器编译的代码等。JDK 8+ 从逻辑上讲，该区域是由的 元空间（Metaspace） 实现，使用本地内存而不是堆积内存；常量池运行（Runtime Constant Pool）作为其中的一部分，也在这里。

直接内存（Direct Memory）：不受 JVM 直接管理内存模型
它不属于运行时数据区的一部分，但经常用于 NIO 堆外缓存(如 ByteBuffer.allocateDirect）。虽然不归 JVM 但受本机总内存和管理的限制 -XX:MaxDirectMemorySize 超限会触发参数 OOM。



补充说明：字符串常量池和运行常量池的位置变化
JDK 7 以后，字符串常量池（String Table）它已经从方法区(永久代)转移到堆中；方法区(元空间)运行时常量池中仍保留类符号引用和数字面量。这意味着 intern() 字符串对象本身就在堆上，引用记录在堆中的字符串常量池。

找出哪些区域线程是私有的，哪些是共享的，可以帮助你准确判断变量生命周期，锁的作用范围，GC 能否回收某个对象，为什么？ static 所有线程都会看到变量——本质上，因为它们都落在共享堆或方法区。
