用 Collections.sort() 对学生成绩对象的排序最直接
Java 实现成绩排名的核心是根据总分或单科分数对学生数据进行排序。只要 Student 类实现 Comparable 接口并重写 compareTo()，就能直接用 Collections.sort() 原地排序。
常见的错误是不处理分数相等时的稳定性——例如，两个学生的总分都是 95，但希望按学号排在前面。这时， compareTo() 补充二级比较逻辑：public int compareTo(Student o) {
int scoreDiff = Integer.compare(o.getTotalScore(), this.getTotalScore()); // 降序：o在前
if (scoreDiff != 0) return scoreDiff;
return Integer.compare(this.getId(), o.getId()); // 当得分相同时，按id升序
}注意：Integer.compare(a, b) 比 a - b 安全，避免整数溢出；排序字段值 null 时会抛 NullPointerException，需要提前验证或使用 Comparator.nullsLast()。

排名序号不仅要依靠索引，还要处理平行情况
排序后直接使用 for (int i = 0; i  赋值 rank = i + 1 这是错误的——它不能反映真实的排名规则(即中国排名/密集排名)“并列第二，下一个是第四”。
正确的做法是手动维护当前的排名和上一名的分数：

初始化 currentRank = 1，lastScore = list.get(0).getTotalScore()

从第 0 位开始循环，每次比较当前的分数和 lastScore

相等则沿用 currentRank；不等则更新 currentRank = index + 1(或更严格:+1 然后和前一名比较)

更强大的写作方法是记录最后一名的排名值，而不是分数，以避免浮点误差或对象引用。

谨慎使用大数据量 ArrayList + 全量排序

数据库层排序：使用 ORDER BY total_score DESC, id ASC，让 MySQL / PostgreSQL 承受排序压力
