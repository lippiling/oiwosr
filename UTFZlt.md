ChromeDriver 启动失败：找不到或版本不匹配
启动 ChromeDriver 报错 java.lang.IllegalStateException: The driver executable does not exist 或 session not created: This version of ChromeDriver only supports Chrome version X，本质是路径不匹配，版本不匹配。
实操建议：
；

别用 System.setProperty(&quot;webdriver.chrome.driver&quot;, &quot;...&quot;) 手动指定路径 —— 改用 WebDriverManager 自动管理（Maven 引入 io.github.bonigarcia:webdrivermanager），调用 WebDriverManager.chromedriver().setup() 即可
Chrome 浏览器更新后，WebDriverManager 默认拉取最新版本的驱动程序；如果需要锁定版本，添加 .driverVersion(&quot;124.0.6367.91&quot;)

如果必须手动放置驱动器，路径中不含中文和空格，并确保 chromedriver 文件有执行权限（Linux/macOS 下用 chmod +x）

找不到元素：findElement 返回 NoSuchElementException
写好 driver.findElement(By.id(&quot;submit&quot;)) 却抛 NoSuchElementException，不是 selector 写错了，大概率是页面还没有加载，元素还没有渲染出来，或者是在 iframe 里。
实操建议：
；

别用 Thread.sleep(3000) 硬等 —— 改用 WebDriverWait 配合 ExpectedConditions，比如 new WebDriverWait(driver, Duration.ofSeconds(10)).until(ExpectedConditions.elementToBeClickable(By.id(&quot;submit&quot;)))

检查是否进入 iframe：用 driver.switchTo().frame(&quot;frame-name&quot;) 切进去再找；切回来用； driver.switchTo().defaultContent()

动态生成的 ID（如 btn-submit-123abc）不要硬写死，改用属性包括匹配:By.cssSelector(&quot;button[id*='submit']&quot;)


点击无效或输入被清空：事件未触发或页面重绘
调了 element.click() 没反应，或 element.sendKeys(&quot;abc&quot;) 输后立即变空很常见 Vue/React 事件绑定延迟、表格验证和拦截，或元素在页面上被绑定 JS 重建。


实操建议：
；

优先用 Actions 模拟真实交互：new Actions(driver).moveToElement(element).click().perform()，比原生 click 更可靠
输入前先清空：element.clear()，再 sendKeys()；若仍被清空，试试 element.setAttribute(&quot;value&quot;, &quot;abc&quot;)（通过 JS 注入）
确认元素是否在视口内：使用： ((JavascriptExecutor) driver).executeScript(&quot;arguments[0].scrollIntoView(true);&quot;, element) 滚动后再操作

爬虫场景被识别为自动化：如何绕过检测
直接用默认 ChromeDriver 访问一些网站（如淘宝、知乎），页面直接跳转到验证页面或返回空内容，因为它们被检测到 webdriver 属性、navigator.webdriver 值，或某些特征指纹。

注入 JS 覆盖关键属性：((JavascriptExecutor) driver).executeScript(&quot;Object.defineProperty(navigator, 'webdriver', {get: () =&gt; undefined})&quot;);
