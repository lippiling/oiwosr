Eclipse中改JDK不生效，确保Installed JREs、Java Compiler compliance level、JRE System Library的版本相同，Maven配置同步；JRE未绑定或绑定错误会导致编译报错。

项目右键 Properties 里改 JDK 不生效？检查 Java Compiler 的 compliance 级别
很多人在 Project Properties → Java Compiler 里调了 Compiler compliance level，但是编译仍然是报纸 lambda expressions are not supported 或 var is not supported —— 这说明 Eclipse 你没必要选择 JDK，只是“假装支持”。compliance level 在不确定实际编译器和操作的情况下，只控制语法校验。它必须和谐 JRE System Library 默认版本和工作区域 JDK、以及 Installed JREs 中对应 JDK 路径三者对齐，否则纸面合规。
实操建议：

先确认 Window → Preferences → Java → Installed JREs 在里面勾选的是你想用的 JDK（比如 jdk-17.0.2），且路径指向 jdk-17.0.2 根目录(不 jre/ 子目录）

Project Properties → Java Compiler 中勾选 Use compliance from execution environment，然后选择相应的环境(如 JavaSE-17）——比手动设 compliance level 更可靠
若项目使用 Maven，pom.xml 中的 maven-compiler-plugin 配置（source/target）会覆盖 Eclipse 设置，需要同步

Build Path 里的 JRE System Library 显示 unbound 怎么办
这是最常见的“”JDK 找不到“信号”，通常是因为项目进口时旧工作区配置残留，或者 Installed JREs 列表里删过 JDK 后没重绑。Eclipse 不会自动帮你重新连接，必须手动触发。
实操建议：

右键项目 → Build Path → Configure Build Path… → 左侧选 Libraries → 展开 JRE System Library → 点击右侧 Remove

再点 Add Library… → 选 JRE System Library → Next → 选 Workspace default JRE(确保这个默认值已经存在 Installed JREs 正确配置)
若要使用特定的 JDK 而非工作区默认，选择 Alternate JRE，从下拉列表中选择已安装的安装列表 JDK 条目(注:列表为空？回去检查 Installed JREs 是否真的成功添加)

多个项目共享一个工作区，如何使每个项目不同 JDK
Eclipse 支持项目级 JDK 绑定，但前提是全局 Installed JREs 所有需要使用的都已经注册 JDK。一旦某个版本被遗漏(例如只有) JDK 11 和 17.我想把项目分配给一个项目 JDK 19），Build Path 找不到选项，强行改变 org.eclipse.jdt.core.prefs 文件也无效。


实操建议：


Window → Preferences → Java → Installed JREs → Add… → 一个接一个地添加你需要的一切 JDK(路径必须完整 JDK 包括安装目录 bin/javac）
右键单击每个项目 → Properties → Java Build Path → Libraries → 替换 JRE System Library 对应版本(例如 jdk-11.0.20 或 jdk-17.0.2）
不要依赖 “Execution Environment” 名称模糊匹配(如 JavaSE-17），某些 JDK 如果安装包没有正确声明执行环境，项目仍然会发生 fallback 到默认 JDK

改完 JDK 编译出错：The type java.lang.Object cannot be resolved

这几乎 100% 是 JRE System Library 绑定不正确 JDK（而非 JRE），或者绑定 JRE(比如只安装 JRE 17，没装 JDK 17）。JRE 不含 rt.jar 或 classes.jsa 等核心编译，Eclipse 找不到 java.lang.* 的源定义。
实操建议：

打开 Build Path → Libraries，展开 JRE System Library，看是否显示 jdk-xx.x.x（带 jdk- 前缀）；若显示 jre1.8.0_301 或类似纯 JRE 名称，立即删除重新绑定
检查 JDK 是否存在安装路径 lib/rt.jar（旧版）或 lib/classes.jsa（JDK 9+）；不存在说明你绑的是 JRE 或损坏安装
重启 Eclipse(有时缓存不刷新，即使设置更改也不生效)

真正的麻烦不是改变哪些菜单，而是改变哪些菜单 Eclipse 会在 Installed JREs、项目 .settings/org.eclipse.jdt.core.prefs、.classpath、Maven 配置、甚至 workspace metadata 里分散存 JDK 信息。只要有一个地方没有对齐，就会出现“看起来变了，其实没用”的情况。
