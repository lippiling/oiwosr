containsall比较调用方集合是否包含参数集合的每一个元素，依靠equals和hashcode实现，将空集返回true，将nullpointerexception扔到null参数。

containsAll 方法的比较是什么？
containsAll 判断的是「调用方集合是否包含参数集合中的**每个元素**」，不严格定义子集关系的数学，也不要求元素的独特性或顺序一致。它的底部调用参数集中的每个元素 contains，而 contains 依赖元素 equals（和 hashCode，若使用哈希类集合)。

对 ArrayList、LinkedList：逐个遍历 + equals，时间复杂度 O(m×n)
对 HashSet、TreeSet：单次 contains 平均 O(1) 或 O(log n)，整体接近 O(m)
若参数集合含有 null，而且不支持目标集合 null（如 TreeSet），会抛 NullPointerException



为什么 ArrayList.containsAll(new HashSet()) 有时返回 false
常见的原因是元素 equals 和 hashCode 实现不一致或不重写。例如，自定义类 User 只重写了 toString，没重写 equals 和 hashCode，然后两个逻辑是相等的 User 实例在 HashSet 它可能被视为不同的对象，但放入其中 ArrayList 后 contains 默认引用失败。

一定要确认：所有参与 containsAll 正确重写了自定义类别 equals 和 hashCode

用 IDE 自动生成(如 IntelliJ 的 Alt+Insert → equals() and hashCode()），不要手写错过字段
测试时用真实对象的实例进行比较，不仅仅依靠 System.out.println 看输出

containsAll 在空集合和 null 上的行为
containsAll 对空集合返回 true(数学上“空集是任意集合的子集”)，但对于 null 参数直接抛 NullPointerException —— 它不接受 null 作为参数。
List list = Arrays.asList("a", "b", "c");
System.out.println(list.containsAll(Collections.emptyList())); // true
System.out.println(list.containsAll(null)); // 抛 NullPointerException

调用前必须判空：if (other != null && list.containsAll(other))

不要依赖 Collections.emptySet() 或 emptyList() 来“兜底”，它们只是不能变空集合，不影响 containsAll 逻辑
若业务允许“空即匹配”，则应单独处理：other == null
