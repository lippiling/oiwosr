用 HashMap 存储学生信息，根据学号快速查看学生信息
学号是唯一稳定的自然主键，HashMap 这是最直接的选择。不要使用它 ArrayList 搜索-100000 个学生查一次可能要经历几百次，但是 HashMap 平均时间复杂度为 O(1)。
注意点：


Student 类必须正确重写 equals() 和 hashCode()，否则，自定义对象作品 value 没问题，但如果将来要用的话 Student 当 key 就会出错
学号作为 key 建议用 String（比如 "2023001"），避免从整数开始 0 时被截断
初始化时不要使用 new HashMap()，改用 new HashMap(16) 或 Map.ofEntries()（Java 9+)预测容量，减少扩容费用

找不到时返回 null 还是抛异常？查看调用场景？

public Optional findStudentByNo(String studentNo) {
return Optional.ofNullable(studentsMap.get(studentNo));

HashMap 只支持精确 key 搜索。假如需要“查姓张的学生”或“名字含‘明’”，就不要遍历 values()——那是 O(n)，而且每次检查都扫描全量数据。

在多线程环境下检查学生，HashMap 没有线程安全
若查询功能正在运行 Web 服务(例如 Spring Boot 的 Controller），同时阅读多个请求 HashMap 看似没事，但只要有人在后台更新(比如引进新生)，就会触发 ConcurrentModificationException 或者读取脏数据。
简单的修复方法：

检查操作比改变操作多得多 → 用 Collections.unmodifiableMap(new HashMap()) 包一层，保证只读视图的安全
需要并发阅读和写作 → 改用 ConcurrentHashMap，但注意它的 get() 尽管线程安全，computeIfAbsent() 是原子操作，不要写 if (!map.containsKey(k)) map.put(k, v)
